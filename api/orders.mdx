The orders API lets you create customer orders and track them through the SmartLogix logistics workflow. When creating an order, the service checks available stock in the inventory service before accepting it. All endpoints require an `Authorization: Bearer <token>` header.

---

## GET /api/pedidos

List all orders.

<CodeGroup>

```bash Request
curl --request GET \
  --url http://localhost:8080/api/pedidos \
  --header 'Authorization: Bearer $TOKEN'
```

```json Response 200

    "creadoEn": "2026-04-20T09:15:00"
  }
```

## POST /api/pedidos

Create a new order. Before saving, the service verifies that the requested product has sufficient stock. If stock is unavailable, the order is rejected.

### Request body

<ParamField body="userId" type="integer" required>
  The numeric ID of the user placing the order.
</ParamField>

<ParamField body="userEmail" type="string" required>
  The email address of the user placing the order. Used for notifications.
</ParamField>

<ParamField body="clienteNombre" type="string" required>
  The name of the end customer receiving the order.
</ParamField>

<ParamField body="total" type="number" required>
  Total monetary amount for the order.
</ParamField>

<ParamField body="tipoPedido" type="string" required>
  Order type. Must be `NACIONAL` or `INTERNACIONAL`.
</ParamField>

<ParamField body="destino" type="string" required>
  Destination address or city for the order.
</ParamField>

<ParamField body="productoId" type="integer" required>
  ID of the inventory product being ordered. Used for stock validation.
</ParamField>

<ParamField body="cantidad" type="integer" required>
  Quantity of the product requested.
</ParamField>

<Note>
Stock is validated before the order is accepted. If `stockActual` for the given `productoId` is less than `cantidad`, the request is rejected with a `500` error and the message `"Stock insuficiente o servicio de inventario no disponible"`. Verify available stock via `GET /api/inventario/{productoId}` before placing the order.
</Note>

### Response fields

<ResponseField name="id" type="integer">
  Auto-assigned order ID.
</ResponseField>

<ResponseField name="userId" type="integer">
  ID of the user who placed the order.
</ResponseField>

<ResponseField name="userEmail" type="string">
  Email of the user who placed the order.
</ResponseField>

<ResponseField name="clienteNombre" type="string">
  Name of the end customer.
</ResponseField>

<ResponseField name="total" type="number">
  Order total amount.
</ResponseField>

<ResponseField name="tipoPedido" type="string">
  `NACIONAL` or `INTERNACIONAL`.
</ResponseField>

<ResponseField name="destino" type="string">
  Destination address.
</ResponseField>

<ResponseField name="status" type="string">
  Current order status. Initial value is `PENDIENTE`. See [order status lifecycle](#order-status-lifecycle) below.
</ResponseField>

<ResponseField name="observaciones" type="string">
  Optional notes attached to the order.
</ResponseField>

<ResponseField name="creadoEn" type="string">
  ISO 8601 timestamp of when the order was created.
</ResponseField>

<CodeGroup>

```bash Request
curl --request POST \
  --url http://localhost:8080/api/pedidos \
  --header 'Authorization: Bearer $TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{
    "userId": 42,
    "userEmail": "warehouse@smartlogix.com",
    "clienteNombre": "Distribuidora del Norte",
    "total": 750.00,
    "tipoPedido": "NACIONAL",
    "destino": "Medellín, Colombia",
    "productoId": 1,
    "cantidad": 50
  }'
```

## Order status lifecycle

Orders move through the following states:

```
PENDIENTE → APROBADO → EN_ENVIO → ENTREGADO
                ↓
            CANCELADO
```

| Status | Description |
|---|---|
| `PENDIENTE` | Order created; awaiting approval. Initial state. |
| `APROBADO` | Order approved and ready for shipment. |
| `EN_ENVIO` | A shipment has been created and is in transit. |
| `ENTREGADO` | Order delivered to the customer. |
| `CANCELADO` | Order was cancelled before delivery. |
