The SmartLogix API is a REST API served through a single gateway at `http://localhost:8080/api`. All requests to protected resources must include a valid JWT in the `Authorization` header. The API covers four resource groups: authentication, inventory management, order processing, and shipment tracking.

## Base URL

```
http://localhost:8080/api
```

All endpoints described in this reference are relative to this base URL.

## Authentication

SmartLogix uses JWT (JSON Web Token) bearer authentication. Include your token in every request to a protected endpoint:

```
Authorization: Bearer <your-token>
```

Obtain a token by calling `POST /api/auth/login` with your email and password. The response body is the raw JWT string. Store it and pass it as the `Authorization` header for all subsequent requests.

A `401 Unauthorized` response means either the header is missing, the token is malformed, or the token has expired. Re-authenticate via `/api/auth/login` to get a new token.

<Note>
The `/api/auth/register` and `/api/auth/login` endpoints are public ‚Äî they do not require an `Authorization` header.
</Note>

## Resources

| Resource | Base path | Description |
|---|---|---|
| Authentication | `/api/auth` | Register and log in to get a JWT token |
| Inventory | `/api/inventario` | Manage products, stock levels, and alerts |
| Orders | `/api/pedidos` | Create and track customer orders |
| Shipments | `/api/envios` | Assign carriers and track shipment status |

## Error responses

| Status | Meaning |
|---|---|
| `400 Bad Request` | The request body failed validation. Check the field constraints. |
| `401 Unauthorized` | The `Authorization` header is missing or the token is invalid. |
| `404 Not Found` | The requested resource does not exist. |
| `500 Internal Server Error` | An unexpected server-side error occurred. |
