This guide walks you through cloning the repository, starting all services, creating an account, and querying the inventory API with a real JWT token. The entire platform runs locally via Docker Compose — no cloud account required.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Docker** and **Docker Compose** (Docker Desktop includes both)
- **curl** or an API client such as Postman or Insomnia

## Setup

<Steps>
  <Step title="Clone the repository">
    Download the SmartLogix source code to your machine:

    ```bash
    git clone https://github.com/emiledubois/fullstack3.git
    cd fullstack3
    ```
  </Step>

  <Step title="Start all services">
    Build and start every service — including all six microservices, the frontend, and the PostgreSQL databases — with a single command:

    ```bash
    docker-compose up --build
    ```

    The first build takes a few minutes while Docker downloads base images and compiles the Java services.
  </Step>

  <Step title="Wait for services to be ready">
    Watch the logs until all containers report that they are running. The two endpoints you'll use most are:

    | Endpoint                     | Description                        |
    | ---------------------------- | ---------------------------------- |
    | `http://localhost:8080`      | API gateway — all API calls go here |
    | `http://localhost:5173`      | React dashboard (browser UI)       |

    <Note>
      The React dashboard is available at [http://localhost:5173](http://localhost:5173). You can use it alongside the API or independently.
    </Note>
  </Step>

  <Step title="Register a user account">
    Create an account by sending your email and password to the auth endpoint:

    ```bash
    curl -s -X POST http://localhost:8080/api/auth/register \
      -H "Content-Type: application/json" \
      -d '{"email": "user@example.com", "password": "secret"}'
    ```

    A successful registration returns a confirmation message. You only need to do this once.
  </Step>

  <Step title="Log in and capture your JWT token">
    Exchange your credentials for a JWT token, which you'll attach to every subsequent request:

    ```bash
    TOKEN=$(curl -s -X POST http://localhost:8080/api/auth/login \
      -H "Content-Type: application/json" \
      -d '{"email": "user@example.com", "password": "secret"}')

    echo $TOKEN
    ```

    The response body is the raw JWT string. The command above stores it in the `TOKEN` shell variable for use in the next step.
  </Step>

  <Step title="Make your first authenticated request">
    List all inventory items by including the token in the `Authorization` header:

    ```bash
    curl -s http://localhost:8080/api/inventario \
      -H "Authorization: Bearer $TOKEN"
    ```

    A fresh installation returns an empty array `[]`. You can add items using `POST /api/inventario`.

    **Example response:**

    ```json
    []
    ```
  </Step>
</Steps>

## Service health checks

You can verify that each backend service is running by calling its health endpoint through the gateway:

```bash
# Inventory service
curl http://localhost:8080/api/inventario/health

# Orders service
curl http://localhost:8080/api/pedidos/health

# Shipments service
curl http://localhost:8080/api/envios/health
```

A healthy service responds with HTTP `200`.

<Warning>
  Health endpoints do not require authentication, but all other API endpoints do. Requests without a valid `Authorization: Bearer <token>` header return HTTP `401`.
</Warning>
